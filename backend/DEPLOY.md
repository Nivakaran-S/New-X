# HealPlace — Production Deployment Guide

## Architecture

| Service | Platform | Free tier |
|---------|----------|-----------|
| API (NestJS) | Render | ✅ 750 hrs/month |
| PostgreSQL | Render | ✅ 90-day free, then $7/mo |
| Redis | Upstash | ✅ 10k cmds/day |
| Search | Meilisearch Cloud | ✅ 100k docs, 10k searches/mo |
| Web store | Vercel | ✅ Unlimited |
| POS terminal | Vercel | ✅ Unlimited |
| Admin panel | Vercel | ✅ Unlimited |
| File storage | Cloudflare R2 | ✅ 10 GB free |
| Email | Resend | ✅ 3k emails/mo |

> **Render free tier note:** Services sleep after 15 min of inactivity and have a ~30s cold start on first request. Upgrade to Starter ($7/mo) to keep it always-on. The PostgreSQL free instance **expires after 90 days** — export your data and recreate (or upgrade) before then.

## Step 1 — Push your code to GitHub

```bash
git init
git add .
git commit -m "Initial commit"
git remote add origin https://github.com/YOUR_USERNAME/healplace.git
git push -u origin main
```

## Step 2 — Set up Upstash Redis (free Redis)

1. Go to [upstash.com](https://upstash.com) → create a free account
2. New Database → choose **Redis** → region: **Asia Pacific (Singapore)**
3. Copy the **Redis URL** — it looks like:
   ```
   rediss://default:TOKEN@HOSTNAME.upstash.io:6379
   ```
4. Keep this for Step 3.

---

## Step 3 — Set up Meilisearch Cloud (free search)

1. Go to [cloud.meilisearch.com](https://cloud.meilisearch.com) → create a free account
2. New project → region: **Asia Pacific**
3. From the project dashboard, copy:
   - **Host URL** (e.g. `https://ms-abc123-sg.meilisearch.io`)
   - **Default Admin API Key**
4. Keep these for Step 4.

---

## Step 4 — Deploy API on Render

### Option A — Blueprint (recommended, one click)

1. Go to [render.com](https://render.com) → New → **Blueprint**
2. Connect your GitHub repo
3. Render reads `render.yaml` and creates:
   - `healplace-api` web service
   - `healplace-db` PostgreSQL database
4. After creation, go to the `healplace-api` service → **Environment** and set the following variables manually (marked `sync: false` in render.yaml):

| Variable | Value |
|----------|-------|
| `REDIS_URL` | Upstash URL from Step 2 |
| `MEILI_URL` | Meilisearch host from Step 3 |
| `MEILI_MASTER_KEY` | Meilisearch API key from Step 3 |
| `WEB_URL` | Your Vercel web URL (set after Step 5) |
| `POS_URL` | Your Vercel POS URL (set after Step 5) |
| `ADMIN_URL` | Your Vercel admin URL (set after Step 5) |
| `R2_ACCOUNT_ID` | Cloudflare R2 account ID |
| `R2_ACCESS_KEY_ID` | R2 access key |
| `R2_SECRET_ACCESS_KEY` | R2 secret key |
| `R2_PUBLIC_URL` | `https://uploads.healplace.lk` |
| `RESEND_API_KEY` | Your Resend API key |

5. `JWT_SECRET` and `JWT_REFRESH_SECRET` are auto-generated by Render (`generateValue: true`).
6. Click **Deploy** — the build installs deps, compiles TypeScript, runs Prisma migrations, and starts the server.

### Option B — Manual setup

1. New → Web Service → connect your GitHub repo
2. **Settings:**
   - Name: `healplace-api`
   - Region: Singapore
   - Runtime: Node
   - Root Directory: `backend`
   - Build Command:
     ```
     npm install && npx prisma generate && npm run build
     ```
   - Start Command:
     ```
     npx prisma migrate deploy && node dist/main.js
     ```
   - Plan: Free
3. Add all environment variables from the table above (plus auto-link the PostgreSQL database).

### Seed initial data

After first deploy, open a **Shell** tab in your Render service:
```bash
node -e "require('./dist/prisma/seed')" 
# or: npx ts-node prisma/seed.ts
```
This creates the owner account (`owner@healplace.lk` / `Admin@1234`), pricing tiers, vehicle capacities, and sample product.

### Your API URL

After deploy: `https://healplace-api.onrender.com`
Health check: `https://healplace-api.onrender.com/api/v1/health`

## Step 5 — Deploy frontends on Vercel

You need **3 separate Vercel projects** (one per app), all pointing to the same GitHub repo.

### For each app (repeat 3 times)

1. Go to [vercel.com](https://vercel.com) → New Project → Import your GitHub repo
2. **Configure project:**

   | App | Root Directory | Project name |
   |-----|---------------|--------------|
   | Web store | `frontend/web` | `healplace` |
   | Admin panel | `frontend/admin` | `healplace-admin` |
   | POS terminal | `pos` | `healplace-pos` |

3. **Build & Output Settings** (override defaults):
   - Install Command: `pnpm install --frozen-lockfile`
   - Build Command: `pnpm build`
   - Output Directory: `.next`

4. **Environment Variables** — each app has its own `.env.example` to follow:

   **Web store** (`frontend/web/.env.example`):
   | Variable | Value |
   |----------|-------|
   | `NEXT_PUBLIC_API_URL` | `https://healplace-api.onrender.com` |
   | `NEXT_PUBLIC_WEB_URL` | `https://healplace.lk` |
   | `NEXT_PUBLIC_META_PIXEL_ID` | Your Meta Pixel ID |
   | `NEXT_PUBLIC_GA_MEASUREMENT_ID` | Your GA4 Measurement ID |
   | `NEXT_PUBLIC_GOOGLE_PLACES_API_KEY` | Your Google Places key |

   **Admin panel** (`frontend/admin/.env.example`):
   | Variable | Value |
   |----------|-------|
   | `NEXT_PUBLIC_API_URL` | `https://healplace-api.onrender.com` |
   | `NEXT_PUBLIC_WEB_URL` | `https://healplace.lk` |

   **POS terminal** (`pos/.env.example`):

   | Variable | Value |
   |----------|-------|
   | `NEXT_PUBLIC_API_URL` | `https://healplace-api.onrender.com` |

5. Click **Deploy**

### After all 3 apps are deployed

Go back to your Render `healplace-api` service → Environment and update:
- `WEB_URL` → your actual Vercel web URL (e.g. `https://healplace.vercel.app`)
- `POS_URL` → your actual Vercel POS URL
- `ADMIN_URL` → your actual Vercel admin URL

Then trigger a redeploy of the API (the CORS settings need the real URLs).

## Updating & redeploying

Render and Vercel both auto-deploy on every `git push` to `main`. To force a redeploy without a code change, use the "Redeploy" button in the dashboard.

Database migrations run automatically on each API deploy (the start command runs `prisma migrate deploy` before starting the server).

## Troubleshooting

**API cold start is slow (~30s)?**
Expected on Render free tier. Upgrade to Starter ($7/mo) for always-on. Or add a cron job (e.g. via cron-job.org) to ping `/api/v1/health` every 14 minutes to keep it warm.

**CORS errors from the frontend?**
Check that `WEB_URL`, `POS_URL`, and `ADMIN_URL` env vars on the Render API match your actual Vercel URLs exactly (including `https://` and no trailing slash).

**Prisma migrate failing on deploy?**
Make sure `DATABASE_URL` is set correctly. On Render, this is auto-linked from the PostgreSQL service. You can verify in the Shell tab: `echo $DATABASE_URL`.

**Meilisearch sync not working?**
Confirm `MEILI_URL` and `MEILI_MASTER_KEY` are set on Render. The search index rebuilds when products are created/updated via the API events.

**pnpm not found on Render?**
The root `package.json` specifies `"packageManager": "pnpm@9.15.0"` — Render uses Corepack to pick this up automatically. If it fails, add this to the build command prefix: `npm install -g pnpm@9 &&`
